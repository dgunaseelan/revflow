Research Findings
Starter Suite ($25/user/mo) — No API access. Limited to 10 users. Cannot support RevFlow's API-driven backend. UNVIABLE.

Pro Suite ($100/user/mo) — 5,000 API calls/day. Has Flows, custom objects, Account access, AppExchange. May be sufficient for low-volume SMBs but needs load-testing validation against RevFlow's usage patterns.

Enterprise ($165-175/user/mo) — Unlimited API calls. Advanced workflow automation, territory management. Safest option but highest cost per org.

Open: Is there a lightweight SKU or internal-use edition that can be provisioned at reduced cost? Does SF Go support zero/reduced-cost provisioning for RevFlow?

Who subsidizes the Salesforce org infrastructure cost for free-tier customers? At scale (10K+ free users), this is the most expensive unresolved business question.
Options Under Consideration
(a) Salesforce subsidizes as customer acquisition cost
(b) Loss-led for a capped period (e.g. 90 days)
(c) Lightweight org edition to reduce per-org cost
(d) SF Go mechanism for zero/reduced-cost provisioning
Architecture Provisioning SLA
What is the Signup API provisioning latency? If org creation takes minutes, the pre-warmed pool model is mandatory for a self-serve freemium experience. Pool management adds operational complexity.
Architecture RevFlow Middleware Server — Scope & Hosting
The RevFlow Middleware Server orchestrates provisioning (org creation, user setup, Slack wiring) but is not in the runtime path — Slack calls SF APIs directly via MCP at runtime. Key questions: Where is it hosted? What is its availability SLA? How does it manage the pre-warmed org pool? What secrets/credentials does it store?

Scope: Provisioning-only server. Handles signup → org creation → integration user setup → Slack workspace connection. Not involved in day-to-day RevFlow operations.

Technical View

Behind the Scenes
Integration User (API-Only)

The RevFlow Middleware creates a Salesforce Integration User with the Minimum Access – API Only Integrations profile. This user:

  • Cannot log in to Salesforce UI
  • Accesses data only via REST, SOAP, or Bulk API
  • Uses OAuth Client Credentials Flow for auth
  • Gets permissions via Permission Sets (not the profile)
JIT User Provisioning

The Middleware maps Slack workspace members to SF users via Just-In-Time (JIT) provisioning. Slack email → SF User. No SF login credentials are ever created for end users — at runtime, Slack calls SF APIs directly via Salesforce and Revenue Cloud MCP.

Architecture Keeping Salesforce Invisible
How do we prevent any Salesforce UI from being accessible to end users?
Research: Integration User + Headless 360
Integration Users with the "Minimum Access – API Only Integrations" profile physically cannot log into the Salesforce UI. The profile is locked and non-editable.

This is now further validated by Salesforce Headless 360, announced April 15, 2026 at TrailblazerDX. Parker Harris asked: "Why should you ever log into Salesforce again?"

Headless 360 exposes the entire Salesforce platform as 60+ MCP tools and APIs — no browser required. This is the strongest architectural validation for RevFlow's invisible-Salesforce approach.

Three-layer invisibility:
1. API-Only Integration Users (no UI login)
2. Slack calls SF APIs via Salesforce & Revenue Cloud MCP (no SF URLs exposed)
3. Headless 360 platform direction (API is the UI)

RevFlow Impact: Slackbot Only Available on Business+ and Enterprise+

Slackbot (Slack's personal AI agent) is only available on Business+ ($15/user/mo) and Enterprise+. Free and Pro plans do not include it. If RevFlow leverages Slackbot for AI-powered revenue workflows, customers on Free/Pro will not have access to those features.

Business+ has a hard limit of 15 Slackbot conversations per week per user. For high-volume sales teams running multiple quotes, approvals, and deal queries daily, this cap could be restrictive. Enterprise+ has unlimited Slackbot usage.

However: RevFlow as a Seamless App works on all Slack plans (Free, Pro, Business+, Enterprise+). The Slackbot limitation only applies if RevFlow's AI features are delivered through Slackbot rather than through RevFlow's own app interface (Block Kit modals, messages, slash commands). This is an important architectural distinction.

Open Question: Slackbot vs. Seamless App for AI Features

Should RevFlow's AI capabilities (quote generation, deal analysis, pricing recommendations) run through Slackbot (limited to Business+/Enterprise+, capped at 15/week on Business+) or through RevFlow's own Seamless App (available on all plans, no conversation caps)? This decision determines the minimum Slack plan requirement for RevFlow's full feature set.

RevFlow: Which Base Edition + Add-On?

RevFlow needs a base Sales Cloud (or Service Cloud) edition plus optionally a Revenue Cloud add-on SKU. The key tradeoffs:

  • Starter Suite ($25) is UNVIABLE — No API access. RevFlow is entirely API-driven; without REST/SOAP API, it cannot function. Max 10 users is also a hard limit.
  • Professional (~$80) is UNVIABLE — API access is not included by default and requires an add-on purchase. Automation is limited in scope and volume. Custom objects are capped. This edition is also being sunset in favor of Pro Suite — not a forward-looking choice for new provisioning.
  • Pro Suite ($100) is VIABLE — Has API access at 5,000 calls/day, which is sufficient for typical RevFlow usage. Replaces Professional with better customization, Flows, and custom objects. Key open question: can Revenue Cloud Growth (RCG) add-on be enabled on Pro Suite? If yes, Pro Suite + RCG at $250/user/mo is a strong cost-effective option.
  • Enterprise ($175) is the SAFE base — Unlimited API calls, advanced workflow automation, territory management, full customization, Einstein AI. The reliable choice for RevFlow's API-heavy backend.
  • + Revenue Cloud Growth ($150 add-on) — Adds native CPQ, configurator, advanced product catalog, subscriptions, and order capture. Purpose-built for what RevFlow does, but brings total to $325/user/mo (with Enterprise base) or $250/user/mo (with Pro Suite base).
  • Enterprise without RC add-on ($175) — May have enough native quoting + product catalog for RevFlow's initial feature set without the RC add-on cost. Needs validation on which RC-specific features RevFlow actually requires.
Open Question: Base Edition + Revenue Cloud Add-On Strategy

Revenue Cloud is not standalone — it requires a Sales or Service Cloud base license. Two key questions: (1) Can Revenue Cloud Growth be enabled on Pro Suite ($100)? If yes, Pro Suite + RCG at $250/user/mo is a cost-effective path. If RCG requires Enterprise, the floor is $325/user/mo. 1. RevFlow Freemium — Slack Plan Eligibility
Needs Decision
Should RevFlow's freemium offering be limited to Slack Business+ and Enterprise+ customers only?
Why This Matters
RevFlow's AI-powered features (quote generation, deal analysis, pricing recommendations) depend on whether they run through Slackbot or through RevFlow's own Seamless App. This determines the minimum Slack plan for RevFlow.

Option A: Freemium for Business+ & Enterprise+ only
  • Business+ ($15/user/mo) — Slackbot with 15 AI conversations/week per user. SAML SSO, SCIM user management, AI search, daily recaps, AI workflow generation.
  • Enterprise+ (Contact Sales) — Unlimited Slackbot. Unlimited workspaces, native DLP, enterprise search across apps, custom templates, EMM integration.
  • Advantage: Higher-value customer segment. More likely to convert to paid RevFlow. Slackbot AI features available to all users.
  • Risk: Excludes the majority of Slack workspaces (Free + Pro). Limits RevFlow's TAM and viral adoption.
Option B: Freemium for all Slack plans (Free, Pro, Business+, Enterprise+)
  • RevFlow as a Seamless App works on all Slack plans — no Slackbot dependency. Block Kit modals, messages, and slash commands have no plan restrictions.
  • Free plan limits: 90-day message history, 10 app integrations, no workflows, 1:1 huddles only.
  • Pro plan ($7.25/user/mo): Unlimited message history, unlimited apps, unlimited workflows, Slack Connect. No Slackbot.
  • Advantage: Maximum reach. Viral adoption via Slack's existing install base. AI features delivered through RevFlow's own app (no caps).
  • Risk: Free-plan customers have 10-app limit (RevFlow counts as 1). No SSO/SCIM — harder to manage at scale. Lower conversion potential.
Recommendation: Support all plans for core RevFlow features (via Seamless App). Gate premium AI features behind Business+ where Slackbot is available. This maximizes reach while preserving upsell leverage.
Impacts: Go-to-market strategy, pricing tiers, Slack partnership terms

3. How to Keep Salesforce Invisible
Researched

Layer 1 — Integration Users: The "Minimum Access – API Only Integrations" profile physically prevents UI login. The profile is locked/non-editable. OAuth Client Credentials flow for server-to-server only. When audited, Salesforce displays "Access Restricted for API Only Users."

Verdict: Architecturally sound. Integration Users + MCP-based API access + Headless 360 direction all align. No fundamental blocker.
